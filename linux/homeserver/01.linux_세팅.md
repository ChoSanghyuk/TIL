### nginx 설정

#### investindicator.conf

```nginx
server {
	listen 80 default_server;
	listen [::]:80 default_server;
    
	server_name _;
	return 301 https://$host$request_uri;
}

server {
    listen [::]:443 ssl;
    listen 443 ssl;
    server_name lomoninvest.com www.lomoninvest.com;
    
    ssl_certificate /etc/letsencrypt/live/lomoninvest.com/fullchain.pem;
		ssl_certificate_key /etc/letsencrypt/live/lomoninvest.com/privkey.pem;

    location /server/ {
        proxy_pass http://192.168.219.109:3000/;
    
    		# blocked:mixed-content 해결
        proxy_set_header X-Forwarded-Proto https;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location / {
        root /home/lomon/workspace/invest_indicator_client/build/web;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
     location ~ /\.well-known/acme-challenge/ {
    	default_type "text/plain";
    	root /var/www/letsencrypt;
  }

}
```



#### :bulb:  "blocked:mixed-content" error 

- occurs when you're trying to load **HTTP content on an HTTPS page** (or vice versa)

  - https(http)로 요청을 보냈지만 front에서는 backend로 http(https)로 요청을 전달할 때 발생

- 해결

  1. Update your API base URL to use HTTPS

     - 대신, backend와 front 모두 https를 지원하고 있어야 함

  2. Proxy backend requests through Nginx (recommended)

     - you don't necessarily need to adjust backend code

       - the proxy headers alone often solve the mixed-content issue.

     - route API requests through your Nginx server using a reverse proxy

       ```nginx
       server {
           listen 443 ssl;
           server_name {my page};
       
           # ... your SSL and frontend config ...
       
           location /api/ {
               proxy_pass http://127.0.0.1:YOUR_BACKEND_PORT/;
               proxy_set_header Host $host;
               proxy_set_header X-Real-IP $remote_addr;
               proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
               proxy_set_header X-Forwarded-Proto $scheme;
           }
       }
       
       ```

     - **What the proxy headers do:**

       - The headers tell your backend "this request originally came via HTTPS," but your Go app doesn't need to actively check them unless it's doing specific things.

### `auto_launch.sh`

```bash
#!/bin/bash

sleep 10

cd ~/workspace/personal_projects/auto_deploy

for file in ./auto_*; do
  if [ -x "$file" ]; then
  echo "Running $file"
  "$file"
  fi
done

sleep 30
```
