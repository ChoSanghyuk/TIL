# AIDD



## 개요



### AI Coding 의미

- 개념

  - 인공지능을 사용하여 프로그래밍 작업을 자동화하는 과정
  - 코딩 프로세스를 간소화하고 향상시키기 위한 다양한 기능을 제공

- 유형

  - Code to Code

    - 사람이 작성한 코드를 입력 받아 AI가 코드를 생성

    - 코드 완성, 코드 개선, 코드 언어 전환과 같은 작업을 포함

      =>  코딩 프로세스 자동화

  - Text to Code
    - 사람이 자연어로 작성한 텍스트를 입력 받아 AI가 코드를 생성
    - 코드 생성, 코드 검색과 같은 작업을 포함
    - 자연어로 된 인간의 지시를 이해하고 해석함으로써, AI는 텍스트 요구사항을 실행 가능한 코드로 변환에 도움

  - Code to Text
    - 사람이 작성한 코드를 입력 받아 AI가 자연어를 생성해주는 기능
    - 코드 내용을 요약하거나 문서화를 제공하며 코드 이해를 돕는 데 활용



### AIDD 의미

- 설계/개발/테스트 전 공정에서 AI기술을 적용하여 이행 프로세스를 최적화
- AI Coding 도구 + 데이터 기반 지속 학습과 적용으로 개발자와 AI가 Pair Programming으로 일하는 방식입
- AI Coding 도구로 GitHub Copilot, Tabnine과 같은 상용 도구와 LG CNS가 자체 개발한 도구를 활용할 수 있음





### Coding AI vs AI Coding

- Coding AI
  - 이미 작성한 코드 분석, 수정 => 추가적인 코드 제안
- AI Coding
  - 코드 직접 작성



### AI Coding 효과

- 자동화

  - 반복 코드 자동 생성

  - 오류 확인 및 코드 개선 

    => 생산성 향상

- 가속화

  - 전반적인 소프트웨어 개발 기간 단축
  - 루틴 작업 축소

- 최적화

  - 성능 개선 포인트 제안
  - 코드 실행 시간 단축

- 품질향상

  - 오류 자동 검사 및 수정 방법 제시
  - 개발표준 준수로 코드 일관성 유지
  - 보안 취약점 식별로 안전한 코드 작성
  - 리팩토링 제안으로 코드 가독성/유지보수성 향상





## Copilot



### 소개

- 개요
  - OpenAI에서 개발한 AI 기반의 코딩 도우미
  - GitHub 코드베이스를 분석하여 사용자 코드에 대한 제안과 완성
  - 문서 작성, 주석 작성 등 다양한 코딩 관련 작업에도 활용 가능
- 단축키 (Windows/Linux)
  - 다음 제안으로 넘기기 : `Alt + ]`
  - 이전 제안으로 넘기기 : `Alt + [`
  - 이번 제안 수락 : `Tab`
  - 10개 제안 한번에 보기 : `Ctrl + Enter`





### 주요 기능

- 코드 제안
  - 함수명 작성 시 구현 코드에 대한 제안이 표시됨

- 단위 테스트 코드 작성
  - 소스 코드에서 주석으로 `함수형.test.js`와 같은 형식으로 입력
  - 주석 라인에서 Ctrl + Enter 키를 입력하면  Github Copilot 탭에 테스트 코드 제안

- 소스 코드 간 변환
  - 정의
    - 작성된 소스 코드를 기존과 다른 프로그래밍 언어로 변환

  - 목적
    - 언어 호환성 개선
    - 코드 재사용
    - 언어 특징 활용

  - 사용
    - VS Code 좌측의 Copilot Chat 아이콘 클릭
    - 작선된 코드 선택 후 입력창에 "000으로 변환해줘" 입력

- CRUD 코드 작성
  - 주석으로 `CRUD API 호출` 입력하고 Ctrl + Enter 키 입력
  - 제안

- 샘플 데이터 생성
  - 샘플 데이터
    - 개요
      - 개발, 테스트, 데모 목적으로 사용되는 데이터의 집합
      - 실제 운영 환경에서 발생할 수 있는 다양한 시나리오를 모방하기 위해 고안 됨
      - 실제 사용자 데이터와 유사한 구조와 특성을 가지고 있지만, 실제 사용자 정보를 포함하지 않는 가공된 데이터

    - 필요성

      - 개발 및 테스트
        - 새로운 기능 개발 또는 기존 기능 수정 시 샘플 데이터 사용

      - 데모 및 프리젠테이션
        - 소프트웨어의 기능을 시영하거나 새로운 사용자를 교육할 때 사용

      - 성능 측정
        - 샘플 데이터를 사용하여 다양한 부하 테스트 및 성능 테스트 수행

      - 보안 및 개인정보 보호
        - 실제 데이터를 보호하면서 개발 및 테스트 작업 수행

  - 사용
    - 주석으로 필요한 샘플 데이터에 대한 설명 입력 후 Ctrl + Enter 키 입력




### Copilot-X 활용

- 소개
  - 개요
    - GTP-4가 적용된 업그레이된 버전
  - 기능
    - Copilot Chat
      - IDE 환경에서 ChatGPT와 동일한 경험을 제공
      - IDE와 긴밀하게 결합하여 코드, 오류 메시지 등을 인식하여 동작
    - Copilot voice
      - Copilot chat을 키보드 없이 음성으로 사용
    - Copilot for Pull Requests
      - PR과 관련된 설명을 대신 작성해 주는 기능
      - PR 작성 중 코드 변경 사항을 검토해서 Comment 내용을 제안할 수 있음
    - Copilot for Docs
      - 대화형 인터페이스를 통해 질의를 하면 기술 문서를 참고하여 답변 또는 질의에 해당하는 문서의 위치를 제공
    - Copilot for CLI
      - CLI 상에서 질문을 통해 명령어를 생성/변경 또는 명령어에 대한 설명을 요청하고 제안 받을 수 있음
- Copilot Chat
  - 개요
    - 채팅 인터페이스이고, 지원되는 IDE 내에서 구문, 프로그래밍 개념, 테스트 케이스, 디버깅 등 다양한 코등 관련 질문에 대한 답변을 받을 수 있음
    - 에디터 내 채팅 인터페이스 제공
    - 개발자의 코드 밍 오류를 인식, 깊이 있는 설명, 단위 테스트 생성, 버그 수정 제안
    - 코딩 이외의 질문에는 답 X
    - 지원 IDE : VS code, Visual Studio, JetBrains
  - 활용
    - 단위 테스트 케이스 생성
      - 편집기에서 열린 코드나 선택한 영역을 기반으로 단위 테스트 케이스 작성
    - 코드 설명
      - 코드의 기능 및 목적에 대한 자연어 설명을 생성
    - 코드 수정 제안
    - 코드 질의 응답
  - 제약 사항
    - 제한된 범위
      - 복잡한 코드 구조를 처리하지 못할 수 있음
      - 공개 리포지토리에 작성된 양에 따라 언어별 제공 수준 상이
    - 잠재적 편향
      - 특정 프로그래밍 언어 또는 코딩 스타일에 편향되어 최적화되지 않았거나 불안전한 코드 제안 가능성 있음
    - 보안 위험
      - 민감한 정보나 취약점 노출 가능성 존재
    - 부정확한 코드
